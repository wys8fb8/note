 一、按作者归属

  1. 无作者的画
     例如平台精选、公版画、系统导入作品，artist_id 或 artist.user_id 为 0。
  2. 有作者且作者资料完整的画
     作品绑定了作者，并且作者公开详情可查询。
  3. 有作者但作者资料缺失的画
     作品里有 artist_id / artist.user_id，但作者详情不存在、下架、审核未通过或接口查不到。

  二、按价格/获取方式

  4. 免费画
     用户不需要支付，通常是加入图库。
  5. 付费画
     用户需要下单购买，通常进入购物车或直接购买。
  6. 价格为 0 但不是免费画
     这类要单独确认，可能是配置错误、活动价、测试数据，不能简单按价格当免费画。

  三、按链上/确权状态

  7. 不上链的画
     作品本身不走链上确权或 NFT 铸造流程。
  8. 待铸造的画
     作品需要上链，但当前还没有完成铸造，例如 not_minted、pending、minting。
  9. 已上链的画
     作品已经完成链上确权/铸造，并且有真实链上编号、证书、交易等数据。
  10. 铸造失败的画
     作品尝试过铸造但失败，需要重试、退款或人工处理。

  四、按库存/发行状态

  11. 不限量画
     没有总发行量限制，或 total_supply 为 0 / 空且业务定义为不限量。
  12. 限量未售罄画
     有总发行量，且 sold_count 小于 total_supply。
  13. 限量已售罄画
     有总发行量，且 sold_count 等于或大于 total_supply。
  14. 库存数据异常画
     例如 sold_count 大于 total_supply、available_count 为负数、total_supply 缺失但又是付费限量画。

  五、按当前用户关系

  15. 游客看到的画
     当前未登录。
  16. 登录普通用户看到的画
     登录人不是作者，也未拥有该画。
  17. 登录且已拥有该画
     登录人已经购买或已加入图库。
  18. 登录且是该画作者
     登录人就是作品作者。
  19. 登录但权限关系异常
     例如后端同时返回“是作者”和“已购买”，或 owned 状态与订单/图库不一致。

  六、按作品可见/业务状态

  20. 上架中的画
     status 为 on_sale / published / active 等可公开访问状态。
  21. 未上架的画
     草稿、审核中、待发布。
  22. 已下架的画
     off_shelf / inactive 等。
  23. 已删除或软删除的画
     deleted_at 有值或等价删除状态。
  24. 状态异常但仍被访问到的画
     详情页能打开，但状态不应该对公众可见。
